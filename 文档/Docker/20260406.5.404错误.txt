我的vue3项目在本地调试模式下一切正常，而在rocker中，
页面内的一切功能正常，但点击浏览器自带的刷新按钮后，会报出404错误

这是一个非常经典的 单页面应用（SPA）路由模式 问题。
为什么会出现这个问题？
在 Vue3 中，如果你使用了 History 模式（即 URL 里没有 # 号），
路由跳转是由 JavaScript (Vue Router) 在浏览器前端完成的，并没有真正向服务器请求新页面。
    . 本地开发时：vite 或 webpack-dev-server 会自动处理这种情况，当你刷新时，它会默默地重定向回 index.html。
    . Docker/生产环境下：你通常会使用 Nginx 来托管静态文件。当你访问 example.com/login 并点击刷新时，
    浏览器会真地向服务器请求 /login 这个文件。但 Nginx 目录下只有 index.html，没有 login 这个文件或文件夹，于是就报了 404。

解决方案：修改 Nginx 配置
你需要修改 Docker 镜像中 Nginx 的配置文件，告诉它：“如果找不到用户请求的资源，请统一返回 index.html，把路由控制权交给 Vue。”

1. 创建一个 nginx.conf 文件
在你的项目根目录下新建一个 nginx.conf：
    Nginx
        server {
            listen       80;
            server_name  localhost;

            location / {
                root   /usr/share/nginx/html;
                index  index.html index.htm;

                # 核心配置：尝试寻找资源，如果找不到就返回 index.html
                try_files $uri $uri/ /index.html;
            }

            error_page   500 502 503 504  /50x.html;
            location = /50x.html {
                root   /usr/share/nginx/html;
            }
        }

2. 修改你的 Dockerfile
确保在构建镜像时，把这个自定义的配置文件拷贝进去：
Dockerfile
    # 构建阶段 (Build stage)
    FROM node:20 as build-stage
    WORKDIR /app
    COPY package*.json ./
    RUN npm install
    COPY . .
    RUN npm run build

    # 运行阶段 (Production stage)
    FROM nginx:stable-alpine as production-stage
    # 拷贝构建后的静态文件
    COPY --from=build-stage /app/dist /usr/share/nginx/html
    # 拷贝刚才写的自定义 nginx 配置
    COPY nginx.conf /etc/nginx/conf.d/default.conf

    EXPOSE 80
    CMD ["nginx", "-g", "daemon off;"]

这样配置后的原理
    1. 用户访问 yourdomain.com/user/profile。
    2. 浏览器向 Nginx 请求 /user/profile。
    3. Nginx 通过 try_files 发现本地并没有这个文件。
    4. Nginx 按照指令返回 index.html 给浏览器。
    5. 浏览器加载 index.html 里的 Vue 代码。
    6. Vue Router 启动，检测当前的 URL 是 /user/profile，自动渲染出对应的“个人中心”组件。

另一个简单的“偷懒”办法
    如果你不想配置 Nginx，最简单的办法是将 Vue Router 的模式改为 Hash 模式（即 URL 变成 example.com/#/login）。
    在你的 router/index.ts 中修改：
        TypeScript
            import { createRouter, createWebHashHistory } from 'vue-router'

            const router = createRouter({
              history: createWebHashHistory(), // 改为 Hash 模式
              routes: [...]
            })
    Hash 模式下，刷新页面不会报错，因为浏览器在请求时会自动忽略 # 后面的内容，永远只向服务器请求根路径，
    所以不会出现 404。不过，History 模式的 URL 看起来更专业。
