Markdown和YAML的区别

简单来说，Markdown 是用来“写文章”的，而 YAML 是用来“写配置”的。
虽然它们长得都有点像“纯文本”，但在开发流程中扮演的角色完全不同。

1. 核心定位对比
特性 | Markdown (.md) | YAML (.yaml / .yml)
    主要用途 | 内容展示。写文档、博客、README。 | 数据传输/配置。设置服务器、定义环境变量。
    侧重点 | 关注样式（标题、加粗、列表）。 | 关注数据结构（对象、数组、键值对）。
    阅读对象 | 主要是人类。 | 主要是程序（但人类读起来也很方便）。

2. 语法风格的区别
    Markdown：标记语言
        它通过符号来告诉浏览器或编辑器如何渲染文本。
        Markdown
            # 这是一个标题
            这是**加粗**的文字。
            * 列表项 A
            * 列表项 B
    YAML：数据序列化语言
        它通过缩进来表示层级关系，严格遵守 key: value 格式。
        YAML
            server:
              port: 8080
              name: "My App"
            features:
              - login
              - upload

3. 它们之间的“亲密关系”：Front Matter
    在很多静态网站生成器（如 Hugo, Jekyll）或笔记软件（如 Obsidian）中，你会看到它们结合使用。

    通常在 Markdown 文件的最顶部，用两个 --- 包围起来的部分就是 YAML，用来定义这篇文章的元数据（标题、日期、标签等），
    而下面的部分才是 Markdown 内容。

    Markdown
        ---
        title: 我的第一篇博客
        date: 2026-04-04
        tags: [Git, 教程]
        ---

        # 正文开始
        这里是我的 Markdown 内容...

总结建议
    . 如果你想写一份让别人读着舒服的说明书，选 Markdown。
    . 如果你想告诉你的 GitHub Action 或者 Docker 容器该怎么运行，选 YAML。

你是在配置 GitHub 的工作流（Workflow）文件时感到困惑吗？通常那种 .yml 文件对缩进的要求非常严格，差一个空格都会报错。
