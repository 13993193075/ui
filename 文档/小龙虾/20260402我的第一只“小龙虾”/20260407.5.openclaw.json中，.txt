openclaw.json中，agents.list的每一个agent元素具体该如何配置？

在 openclaw.json 中，agents.list 数组用于定义系统中存在的每一个“机器人”（Agent）的核心属性。
根据你提供的配置文件，每一个 Agent 元素是一个 JSON 对象，
主要包含身份标识、工作路径、AI 模型配置以及交互行为设定。

以下是 Agent 元素的具体配置字段详解：

1. 基础配置 (必填字段)
这些是定义一个机器人最基本的属性：
    . id (字符串)
        . 作用：机器人的唯一内部标识符。
        . 重要性：这个 ID 是核心枢纽，后续在 bindings 数组中，
            就是通过这个 id 将机器人与具体的 Telegram 或 Slack 账号连接起来的。
        . 示例："id": "game_designer"

    . workspace (字符串)
        . 作用：指定机器人在服务器（或 Docker 容器内）的专属物理工作目录。
        . 用途：机器人的对话记忆、长期存储、生成的临时文件、日志等都会保存在这个路径下。
            不同机器人必须使用不同的路径以防止数据冲突。
        . 示例："workspace": "/home/node/.openclaw/workspace/game_designer"

    . model (字符串)
        . 作用：指定该机器人使用的底层大语言模型。
        . 格式：通常是 提供商/模型ID 的格式，必须与 models.providers 中已配置的模型相匹配。
            例如你的配置中提供了深度思考（deepseek-reasoner）和标准对话（deepseek-chat）。
        . 示例："model": "deepseek/deepseek-reasoner"

2. 身份设定 (identity 对象)
    这个对象决定了机器人的“人设”和外观表现：
    . name (字符串)：机器人的显示名称或称呼（例如 "Aria" 或 "麦穗"）。
    . theme (字符串)：机器人的核心主题或背景设定，这会作为系统提示词（System Prompt）的一部分，
        影响其语言风格和专业倾向（例如 "multidisciplinary innovation lab" 或 "pixel art idle farming game development"）。
    . emoji (字符串)：代表该机器人的专属表情符号（例如 "📈" 或 "🎨"）。

3. 群聊行为 (groupChat 对象)
    这个对象控制机器人在群组环境下的响应触发规则：
    . mentionPatterns (字符串数组)
        . 作用：定义哪些词汇或符号可以“唤醒”该机器人。如果在群聊中，用户的消息包含了数组中的任意一个词，
            机器人就会认为你在和它说话并进行回复。
        . 示例：["@game_designer", "@maisui", "麦穗"]。通常包括它的 @ 账号名、代号或中文名。

4. 主动触发/定时任务 (heartbeat 对象 - 高级/可选)
    从你之前的 stock_market_analyst (Aria) 的配置中可以看到，Agent 还支持定时主动发起任务：
        . every (字符串)：触发的频率时间，例如 "15m" 表示每 15 分钟触发一次。
        . target (字符串)：消息发送的目标平台，例如 "slack" 或 "telegram"。
        . to (字符串)：目标群组的 ID 或频道标识。
        . 作用：非常适合做股票盯盘、定时汇报、天气推送等需要主动“说话”的场景。

完整结构示例
综合以上所有字段，一个配置齐全的 Agent 元素结构如下所示：
JSON
    {
      "id": "agent_unique_id",
      "workspace": "/home/node/.openclaw/workspace/agent_unique_id",
      "model": "deepseek/deepseek-reasoner",
      "heartbeat": {
        "every": "15m",
        "target": "telegram",
        "to": "特定的群组ID"
      },
      "identity": {
        "name": "机器人昵称",
        "theme": "机器人的核心专业领域或人设描述",
        "emoji": "🤖"
      },
      "groupChat": {
        "mentionPatterns": [
          "@telegram_bot_name",
          "昵称"
        ]
      }
    }
