如何从零开始搭建一个简单的 Monorepo 环境

太棒了，我们这就以目前前端最流行的 pnpm + Turborepo 方案为例。这个组合既轻量又极速，是目前大多数新项目的首选。

第一步：初始化工作区
首先，创建一个文件夹并初始化。
Bash
    mkdir my-monorepo && cd my-monorepo
    pnpm init

第二步：配置 pnpm 工作区
在根目录下创建一个 pnpm-workspace.yaml 文件。这个文件告诉 pnpm 你的项目代码都藏在哪里。
YAML
    packages:
      - 'apps/*'    # 存放完整的应用（如 Web 端、管理后台）
      - 'packages/*' # 存放共享的组件、工具函数、配置

第三步：创建项目结构
手动建立这些文件夹，并快速初始化两个子项目：
    1. 共享工具库：packages/utils
    2. Web 应用：apps/web

    Bash
        mkdir -p apps/web packages/utils

        # 初始化 utils
        cd packages/utils && pnpm init
        # 在 utils/index.js 里写个简单的函数：
        # export const add = (a, b) => a + b;

        # 初始化 web
        cd ../../apps/web && pnpm init

第四步：在应用中引用共享库
    这是 Monorepo 的魔法时刻。你不需要把 utils 发布到 npm，直接在根目录执行：
    Bash
        # 将 packages/utils 作为依赖安装给 apps/web
        pnpm add @my-repo/utils --filter web --workspace

    现在，你在 apps/web 的代码里就可以像引用普通 npm 包一样引用 utils 了。

第五步：引入 Turborepo (进阶必选)
当项目变多时，编译会变慢。Turborepo 通过 缓存 解决这个问题。
    1. 安装：pnpm add turbo -Dw
    2. 配置：在根目录创建 turbo.json：
        JSON
            {
              "$schema": "https://turbo.build/schema.json",
              "pipeline": {
                "build": {
                  "dependsOn": ["^build"],
                  "outputs": ["dist/**"]
                },
                "dev": {
                  "cache": false,
                  "persistent": true
                }
              }
            }

核心操作指令总结
    需求 | 指令
        全局安装依赖 (如 TypeScript) | pnpm add typescript -Dw
        给特定项目安装依赖 | pnpm add axios --filter web
        一键启动所有项目 | pnpm turbo dev
        只构建受影响的项目 | pnpm turbo build

避坑小贴士 💡
    . 命名规范：建议给子项目起名时加上前缀，如 @my-project/ui，这样在引用时一眼就能看出是内部库。
    . 版本同步：虽然 Monorepo 允许不同版本，但尽量保持全局依赖（如 React, ESLint）版本统一，否则会遇到奇怪的编译问题。
